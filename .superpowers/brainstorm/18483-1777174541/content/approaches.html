Architectural Approaches

How should the floating meal suggestion panel integrate into the app?

A

Floating Button + Sidebar Panel

Small button (bottom-right) that expands into a fixed sidebar on the same side. Panel persists while user navigates, closes on route change.

Pros

  • Always accessible
  • Dedicated space for meal suggestions
  • No modal overlay clutter

Cons

  • Takes up horizontal space
  • Layout shift when expanding
  • Mobile unfriendly
B

Floating Button + Modal Dialog

Small button (bottom-right) opens a centered modal/dialog. Modal closes both on dismiss and on route change.

Pros

  • No layout disruption
  • Mobile responsive
  • Lightweight implementation

Cons

  • Blocks access to page
  • Closes on navigation
  • Feels more transient
C

Floating Button + Collapsible Drawer (Recommended)

Small button (bottom-right) expands into a sliding drawer panel. Responsive: takes sidebar space on desktop, bottom drawer on mobile. Auto-closes on route change.

Pros

  • Mobile & desktop optimized
  • No modal overlay
  • Accessible, always available
  • Responsive design

Cons

  • Slightly more complex
  • Need responsive logic

Option C (Collapsible Drawer) is recommended — it gives you the best UX across all screen sizes and matches your requirement for bottom-right placement that closes on navigation.
